# Lab prereg — `avoidance_direct` (COMMITTED BEFORE DATA)

**Sponsor:** operator (2026-07-05). The DIRECT test of the house's most
load-bearing, least-examined assumption: *"the LLM's one measured-real skill is
avoidance."* We assert it constantly (Achilles sold-ban, Nemesis veto, Plutus
Lens-B) but have never isolated it. This measures it. Committed before data.

## The question (a meta-test of the instrument, framed as a tradable overlay)

Does **avoidance add return**, and does the **LLM's veto beat a cheap mechanical
distress screen**? If excluding LLM-flagged names raises a basket's return more
than excluding mechanically-flagged names, the LLM avoidance edge is real. If it
only ties the mechanical screen, "the LLM edge" is softer than the house treats
it — and `avoidance_overlay` (#14, the god) is dead on arrival.

## Three arms (the A/B/C)

Over a universe (Plutus top-500 LARGE and a SMALL/MICRO bucket), each period
exclude the flagged names and hold the survivors equal-weight:
- **Arm C — random exclusion** (the null: does removing *any* k% help?).
- **Arm B — mechanical distress screen** (Altman-Z / going-concern language /
  short interest / accruals / net-debt spike — enumerable from SF1 + EDGAR).
- **Arm A — LLM veto** (reads the filings, flags fraud / going-concern /
  covenant-breach / terminal-decline risk).

**Metric — "avoidance alpha":** return of (universe − excluded) minus the full
universe, per arm. The head-to-head is **A vs B vs C**.

## The honest layering (cost reality)

Running the LLM on thousands of names × dates historically is expensive, so:
- **BACKTESTABLE NOW:** Arms B and C — does *mechanical* avoidance beat random?
  This is the necessary precondition. If mechanically excluding distressed names
  does NOT beat random exclusion, avoidance-as-return-source is dead and the LLM
  version is very unlikely to rescue it.
- **FORWARD A/B (paper):** Arm A vs Arm B — the real question (LLM over
  mechanical) — graded going forward on live names, like Plutus's Lens-B.

## Gates

- **G1:** distress flags + filing reads, not price. PASS.
- **G2 — the honest weak point:** the edge is the market *underreacting* to
  deterioration signals (an information-PROCESSING edge), not a structural
  constraint. "Underreaction" is weak G2 and this is a **capability-frontier bet
  that DECAYS with adoption** (house-view doc). Justified anyway: it directly
  tests the LLM's core claimed edge, and avoidance is the one thing the house
  has *any* positive read on (Achilles sold-ban −1.6%/wk).
- **G3:** SCALABLE (applies to the whole book at any size) — the rare
  growth-compatible shape.
- **G4:** fail (avoids drift, no contractual terminal).
- **G5:** strong (whole-universe, every period).

## Success thresholds

**Backtest (Arm B vs C):** mechanical avoidance alpha > 0, in-sample AND holdout,
non-isolated across {LARGE, SMALL} and across sensible exclusion fractions
(k = 5/10/20%). Miss → the whole avoidance thesis is refuted (mechanical can't
even beat random), and #14 dies with it.
**Forward (Arm A vs B):** the LLM veto beats the mechanical screen's avoidance
alpha over ≥20 graded periods. This is the headline — does the LLM *read*
deterioration better than a checklist?

## Pre-committed consequence

Mechanical avoidance works → open the forward A/B; this becomes the measurement
under `avoidance_overlay` (#14). Mechanical avoidance fails → **the house's
"avoidance is the real edge" claim is REFUTED** — a genuinely important negative
that would force honesty about what the LLM actually does. Either way, ledger row.
